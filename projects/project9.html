Heart Rate Calculation from ECG Signals using DSP

This project focuses on extracting heart rate from ECG signals using digital signal processing techniques. It implements the Pan‑Tompkins algorithm for R‑peak detection, allowing accurate heart rate calculation and visualization from pre-recorded ECG data.

Objective

To develop a system for accurately calculating heart rate from ECG signals by detecting R‑peaks and computing R‑R intervals, while providing graphical visualization of the processed signals.

Scope

The system processes ECG data from the MIT‑BIH Arrhythmia Database. It applies filtering, R‑peak detection, and heart rate computation. Limitations include sensitivity to noise and dependency on ECG acquisition hardware quality.

Key Concepts

  • ECG Components: P wave, QRS complex, T wave
  • QRS Complex: Represents ventricular depolarization, most prominent feature
  • R‑Peak: Tallest point in QRS, used for heart rate calculation

Algorithm Overview

The Pan‑Tompkins Algorithm was implemented with the following steps:

  1. Band-pass filtering (0.5–45 Hz) to remove baseline wander, high‑frequency noise, and power line interference
  2. Differentiation to highlight steep slopes of QRS complexes
  3. Squaring to emphasize large differences and suppress noise
  4. Moving window integration (~150 ms) to smooth and highlight QRS complexes
  5. Adaptive thresholding for peak classification

Heart Rate formula: Heart Rate (BPM) = 60 / Average R-R Interval (seconds)

Implementation

  • Tools: Python (NumPy, SciPy, Matplotlib, Pandas, Tkinter), MATLAB (prototyping)
  • Graphical User Interface for loading ECG files, visualizing raw/filtered signals, and displaying heart rate
  • Real‑time zooming and signal navigation

Data Source

MIT‑BIH Arrhythmia Database — 360 Hz sampling rate, containing both normal and abnormal heart rhythms.

Results

  • Raw ECG, filtered ECG, differentiated, squared, and integrated signals plotted
  • Accurate R‑peak detection on test signals
  • Calculated average heart rate displayed in BPM

Strengths & Limitations

  • High accuracy in clean signals
  • Real‑time capable algorithm
  • Sensitive to extreme noise without additional filtering

Conclusion

The system successfully calculates heart rate from ECG signals using DSP methods. Future work includes advanced noise filtering, testing with real‑time ECG hardware, and integrating more robust detection algorithms.
